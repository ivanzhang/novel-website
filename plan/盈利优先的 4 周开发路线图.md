# 盈利优先的 4 周开发路线图

## 目标

未来 4 周的核心目标不是继续把网站做成更完整的阅读平台，而是尽快跑通最小可行的盈利闭环。

这条闭环应当清晰地包含以下环节：

内容曝光 -> 免费试读 -> 付费拦截 -> 支付成交 -> 继续阅读 -> 复购或续费

所有开发任务都应优先服务于这条链路。凡是不能明显提升拉新、转化、成交、留存的事项，都应暂时后置。

## 第 1 周：打通试读到付费的基础链路

### 本周目标

让未登录用户可以进入内容，完成“先试读，再引导注册和付费”的基础转化路径。

### 重点任务

1. 调整访问路径
- 首页允许未登录浏览小说列表
- 小说详情页允许未登录访问
- 可公开展示小说简介、作者、评分、章节目录
- 免费章节允许未登录试读

2. 实现首个正式付费墙
- 在用户进入付费章节时展示拦截页
- 付费墙展示当前权益说明
- 付费墙同时给出“开会员”和“解锁本书”两个入口
- 对未登录用户先提示登录，再回到当前书和当前章节

3. 补充基础转化文案
- 试读结束后的引导文案
- 付费墙标题、副标题、权益说明
- 会员页卖点文案
- 首单优惠提示

4. 增加最基础的埋点
- 首页点击详情
- 详情页开始阅读
- 免费章节阅读完成
- 付费墙曝光
- 会员页访问
- 购买按钮点击

### 交付标准

- 新用户无需登录即可进入试读
- 用户在关键节点会被稳定引导到付费墙
- 能看到最基础的转化漏斗数据

### 本周不要做

- 评论增强
- 评分优化
- PWA
- App
- 推荐系统

## 第 2 周：降低首单门槛，提升首次成交率

### 本周目标

把“只有会员套餐”的付费模式，升级为更适合冷启动阶段的多层次商品体系，重点提升首单转化。

### 重点任务

1. 新增低门槛商品
- 3 天体验卡
- 7 天体验卡
- 首充优惠包
- 单本解锁

2. 调整会员中心结构
- 会员页按“推荐优先级”排序，而不是按月份排序
- 默认高亮首单优惠商品
- 明确展示适用人群，例如“先试 3 天”“只想看这一本”
- 增加权益对比说明

3. 设计分层转化策略
- 新用户优先看到体验卡
- 高阅读深度用户优先看到月卡
- 对某本书高意向用户优先展示单本解锁
- 对多书浏览用户优先展示会员

4. 开始记录收入相关数据
- 各商品点击率
- 各商品下单率
- 首购金额
- 首购商品类型
- 支付成功率

### 交付标准

- 站内至少有 3 类可售商品
- 低门槛商品可正常购买
- 能看出哪种商品更容易成交

### 本周不要做

- 社区体系
- 复杂优惠券系统
- 多级分销

## 第 3 周：做内容运营抓手，提升留存和复购

### 本周目标

让用户不是只买一次，而是持续回来继续阅读。重点围绕更新感、追更感和高转化内容组织来提升留存与复购。

### 重点任务

1. 补充连载运营能力
- 小说增加连载中和已完结状态
- 展示最近更新时间
- 新增“今日更新”区域
- 新增“最近更新”列表
- 连载作品支持下章预告

2. 增加用户留存抓手
- 收藏小说
- 更新提醒入口
- 已购用户继续追更提示
- 阅读页展示“下一章即将更新”或“今日已更新”

3. 首页改成运营页
- 置顶主推题材
- 增加热门榜、畅销榜、上新榜
- 给重点转化书更多曝光位
- 首页从“功能导航页”转成“卖书页”

4. 开始按内容看转化
- 哪本书详情点击率最高
- 哪本书试读完成率最高
- 哪本书付费转化最高
- 哪个题材最能带来首购

### 交付标准

- 用户能明显感知到更新节奏
- 首页具备基本运营能力
- 能按书和按题材观察收入表现

### 本周不要做

- 大规模社交互动模块
- 评论点赞体系
- 用户关注体系

## 第 4 周：完善支付闭环，开始做转化优化

### 本周目标

在有了试读链路、商品体系和内容运营基础后，第四周重点是补齐真正的支付闭环，并开始做第一轮商业化优化。

### 重点任务

1. 接入真实支付能力
- 支付下单
- 支付回调
- 支付成功更新权益
- 订单状态查询
- 补单机制
- 支付失败处理

2. 完善用户权益系统
- 我的会员状态
- 我的已购小说
- 我的订单记录
- 权益有效期展示
- 续费入口

3. 做第一轮转化优化
- 测试不同免费章节数
- 测试不同付费墙文案
- 测试不同商品默认推荐位
- 测试会员和单本解锁的展示顺序

4. 补充必要的合规内容
- 版权来源说明
- 用户协议
- 隐私说明
- 投诉反馈入口

### 交付标准

- 用户可以完成真实支付并自动获得权益
- 后台可以追踪订单状态
- 可以根据数据开始做第一轮调价和改版

## 4 周后的阶段性验收指标

4 周结束后，建议至少检查以下结果：

- 是否已经形成“试读 -> 付费 -> 获得权益 -> 继续阅读”的闭环
- 是否至少有一种低门槛商品能稳定成交
- 是否知道哪本书、哪个题材、哪个价格包转化最高
- 是否具备真实收款和订单追踪能力
- 是否已经能开始基于数据做转化优化

如果这几个问题的答案都是肯定的，说明项目已经从“阅读功能演示站”进入“具备实际盈利能力的内容产品”阶段。

## 建议的执行原则

在这 4 周内，建议团队统一采用以下原则：

- 先做能收钱的功能，再做体验加分项
- 先做低门槛成交，再做高客单价提升
- 先做可验证的数据闭环，再做主观判断的优化
- 先跑通一个高转化题材，再复制到其他内容

这份路线图的核心不是把网站做得更大，而是尽快验证这个网站是否真的能卖出内容、持续收回收入。
